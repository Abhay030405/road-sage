# RoadSage — Model Weights

This directory holds all pretrained and fine-tuned model weights used by the
RoadSage inference pipeline.  **All `.pth` and `.onnx` files are gitignored**
and must be obtained or produced separately (see instructions below).

---

## Required Files

| File | Model | Size (approx.) | Source |
|---|---|---|---|
| `lane_detector.pth` | UFLDv2 ResNet-18 | ~25 MB | Manual — GitHub releases |
| `lane_detector.onnx` | UFLDv2 (exported) | ~25 MB | Generated by `export_onnx.py` |
| `depth_estimator.pth` | MiDaS Small | ~50 MB | Auto via `torch.hub` / manual |
| `object_detector.pth` | NanoDet-Plus-m | ~20 MB | Manual — GitHub releases |
| `decision_cnn.pth` | MobileNetV3-Small | ~10 MB | Trained in Phase 4 |
| `decision_cnn.onnx` | MobileNetV3 (exported) | ~10 MB | Generated by `export_onnx.py` |

## How to Obtain Each Model

### 1. UFLDv2 ResNet-18 — Lane Detector

**Repository:** https://github.com/cfzd/Ultra-Fast-Lane-Detection-v2

1. Go to the [Releases page](https://github.com/cfzd/Ultra-Fast-Lane-Detection-v2/releases).
2. Download `culane_res18.pth` (CULane) or `tusimple_res18.pth` (TuSimple).
3. Place the file at `models/lane_detector.pth`.
4. Export to ONNX:
   ```bash
   python training/scripts/export_onnx.py --model lane_detector
   ```

**Status:** Manual download required.

---

### 2. MiDaS Small — Depth Estimator

**Repository:** https://github.com/isl-org/MiDaS

**Auto-download (online):** Downloaded automatically on first use by `torch.hub`.
No action needed if the runtime machine has internet access.

**Manual (offline / air-gapped):**
1. Go to the [Releases page](https://github.com/isl-org/MiDaS/releases).
2. Download `midas_v21_small_256.pt`.
3. Place it at `models/depth_estimator.pth`.

**Status:** Auto-download on first use (torch.hub); manual option available.

---

### 3. NanoDet-Plus-m — Object / Obstacle Detector

**Repository:** https://github.com/RangiLyu/nanodet

1. Go to the [Releases page](https://github.com/RangiLyu/nanodet/releases).
2. Download `nanodet-plus-m_416.pth`.
3. Place it at `models/object_detector.pth`.

**Status:** Manual download required.

---

### 4. MobileNetV3-Small — Decision CNN

This model is **trained from scratch** during Phase 4 of the RoadSage pipeline
using pseudo-labels generated by the lane detector.  No pretrained checkpoint
needs to be downloaded.

Training is triggered by:
```bash
python training/scripts/train_decision_cnn.py --config configs/production.yaml
```

**Status:** No download needed — trained in Phase 4.

---

## Quick Start

```bash
# Download / verify all available weights (idempotent — safe to run again)
bash models/download_models.sh

# Export lane detector to ONNX after placing the .pth file
python training/scripts/export_onnx.py --model lane_detector

# Export all exportable models at once
python training/scripts/export_onnx.py --model all
```

## Notes

- `.pth`, `.onnx`, `.pt`, `.bin`, and `.ckpt` files are listed in `.gitignore`
  and will **never** be committed to the repository.
- If you accidentally commit a weight file, remove it with:
  ```bash
  git rm --cached models/<filename>
  git commit -m "chore: untrack model weights"
  ```
- For reproducible experiments, record the exact checkpoint filename and its
  SHA-256 hash in your experiment log (e.g. `wandb` run config).
